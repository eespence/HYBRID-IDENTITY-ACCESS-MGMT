# About Me

My name is Edward Spence. I work in identity and access security — the part of cybersecurity responsible for how users authenticate, what they can access, and how administrative control is protected.

I became focused on identity after noticing that many security incidents are not caused by malware, but by authentication problems. When identity systems fail, organizations cannot operate: users are locked out, administrators lose control, and monitoring systems lose reliability. My work centers on understanding those failure conditions and restoring access safely.

I design and operate lab environments that model real enterprise networks. These environments include Active Directory, Microsoft Entra ID integration, privileged access controls, and centralized monitoring. I validate them by testing authentication paths, administrative access, and recovery procedures, including diagnosing and correcting authentication failures.

## Technical Focus

* Hybrid Active Directory and Microsoft Entra ID identity integration
* Privileged access and administrative session control
* Kerberos and directory authentication behavior
* Security monitoring and authentication log analysis using SIEM platforms
* Windows and Linux access control

Every project in my portfolio follows an operational lifecycle:

**Design → Deployment → Validation → Failure → Root Cause → Remediation**

I intentionally document failures and recovery procedures. Real environments do not remain stable indefinitely, and a security engineer must be able to diagnose and restore authentication and access without increasing risk.

Identity functions as the control plane of security. When authentication and authorization are reliable, monitoring, auditing, and incident response become dependable as well.
